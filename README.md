# Копилка — учёт леев (Next.js + Supabase)

Простое приложение для двух детей: начисляешь леи за дела и режим (только плюсы, ничего не отнимается), можно копить «в росте» под 20% в неделю (сложный процент), снимать в любой будний день и выдавать на руки. История и настройки внутри.

## Что нужно
- Node.js 18+ (рекомендую 20)
- Бесплатный проект на [Supabase](https://supabase.com)

## 1. Supabase
1. Создай проект на supabase.com.
2. Открой **SQL Editor → New query**, вставь содержимое `supabase/schema.sql` и нажми **Run**. Это создаст таблицы, функции, стартовых детей (Ева, Серёжа) и список дел.
3. *(Необязательно)* Авто-начисление по понедельникам: **Database → Extensions** включи `pg_cron`, затем раскомментируй последнюю `select cron.schedule(...)` в `schema.sql` и выполни её. Если не включать — не страшно: приложение само догоняет проценты при каждом открытии (функция идемпотентна, лишнего не начислит).

## 2. Ключи
**Project Settings → API**, скопируй `Project URL` и `anon public` ключ. Создай файл `.env.local`:

```
NEXT_PUBLIC_SUPABASE_URL=https://твой-проект.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=твой-anon-ключ
```

## 3. Локально
```bash
npm install
npm run dev
```
Открой http://localhost:3000

## 4. Деплой на Vercel
1. Залей репозиторий на GitHub.
2. В Vercel **Add New → Project**, импортируй репо.
3. В **Environment Variables** добавь те же `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. Deploy.

## Как работает
- **Кнопки начисления** на карточке ребёнка добавляют леи в «Доступно». Список дел и суммы правятся в ⚙︎ настройках.
- **«В рост → / ← Снять»** — перекладывают между «Доступно» и «В росте».
- **Проценты** капают на сумму «в росте» каждый понедельник (по умолчанию 20%, меняется в настройках). Хочешь «забрать проценты» — просто сними их; не снял — они растут дальше (компаундинг).
- Все операции пишутся в историю. «Выдать на руки» обнуляет доступное и отмечает выдачу.

## Безопасность (важно)
В `schema.sql` включён RLS, но политики **открыты для роли `anon`** (чтение и запись). Этого достаточно для приватного деплоя по нерасшариваемой ссылке, но **не** для публичной ссылки — кто угодно с URL и anon-ключом сможет писать.

Чтобы закрыть:
- добавь **Supabase Auth** (например, один общий вход для родителей) и поменяй в политиках `to anon` → `to authenticated`;
- и/или включи **Vercel Password Protection** на проекте;
- не публикуй ссылку.
